# Generated Skill Contract

`install-hve-core-rpiv` generates four local skills from the current HVE Core source commit. Generated skills are derived projections, not maintained catalog source.

## Generated skill set

| Skill | Source prompt | Source agent | Purpose |
|-------|---------------|--------------|---------|
| `task-research` | `.github/prompts/hve-core/task-research.prompt.md` | `.github/agents/hve-core/task-researcher.agent.md` | Research implementation context and produce an evidence-backed research artifact. |
| `task-plan` | `.github/prompts/hve-core/task-plan.prompt.md` | `.github/agents/hve-core/task-planner.agent.md` | Convert research into a plan and detailed implementation guidance. |
| `task-implement` | `.github/prompts/hve-core/task-implement.prompt.md` | `.github/agents/hve-core/task-implementor.agent.md` | Implement from the plan while respecting phase/step stop controls where the runtime supports them. |
| `task-review` | `.github/prompts/hve-core/task-review.prompt.md` | `.github/agents/hve-core/task-reviewer.agent.md` | Review changes against research, plan, and implementation artifacts. |

Do not generate `rpi` in v1.

## Folder shape

For the default destination `.agents/skills`, generated output is:

```text
.agents/skills/
├── .hve-core-rpiv-install.json
├── task-research/SKILL.md
├── task-plan/SKILL.md
├── task-implement/SKILL.md
└── task-review/SKILL.md
```

The manifest sits at the destination root. Generated skill directories are managed files only when listed in the manifest.

## Frontmatter

Every generated `SKILL.md` must include only skill-compatible frontmatter:

```yaml
---
name: task-research
description: Research implementation context using a skill-shaped projection of HVE Core RPIV.
---
```

Allowed required keys:

- `name`
- `description`

Do not copy agent or prompt frontmatter keys into generated skills. These keys are forbidden:

- `agent`
- `agents`
- `handoffs`
- `disable-model-invocation`

## Required generated body sections

Every generated skill body must include:

1. Source disclosure:
   - generated by `install-hve-core-rpiv`
   - HVE Core source repository
   - source ref
   - source commit
   - source prompt path
   - source agent path
2. Non-parity disclosure:
   - this is a skill-shaped projection
   - exact HVE Core agent picker, subagent, handoff, model-selection, and prompt-dispatch semantics may not be preserved
3. Inputs:
   - arguments or context expected by the upstream prompt
   - examples where the upstream prompt provides them
4. Behavior:
   - concise role and constraints from the matching HVE Core agent
   - no independent RPIV behavior invented outside the source prompt/agent
5. Outputs:
   - expected artifact family and handoff to the next RPIV phase
6. Update guidance:
   - rerun `install-hve-core-rpiv` to refresh from HVE Core
   - avoid hand-editing generated managed files unless prepared for drift prompts on rerun

## Expected artifact families

Generated skills should preserve HVE Core's artifact handoff habit:

| Skill | Expected artifact family |
|-------|--------------------------|
| `task-research` | `.copilot-tracking/research/` |
| `task-plan` | `.copilot-tracking/plans/` and `.copilot-tracking/details/` |
| `task-implement` | code changes and `.copilot-tracking/changes/` |
| `task-review` | `.copilot-tracking/reviews/` |

If the target runtime cannot create these exact files, the generated skill must still explain the expected handoff artifact shape.

## Projection limits

Generated skills must not promise:

- native HVE Core agent identity
- automatic subagent execution
- HVE Core prompt frontmatter dispatch
- exact slash-command behavior in every client
- model-selection parity

If a client does not expose generated skills as slash commands, the final installer report should tell the user to invoke them through that client's skill mechanism.
