## Summary

The-flow's 11 stage modules carry 147 cross-stage references because the v1 authoring guide prescribed them — every edge lives in 5 places, so one formatting change cost 308 string edits. This plan ships the shareable fix (the **sub-skill pattern**: a ruleset doc + firm templates + a deterministic lint script) and proves it by refactoring the-flow as the exemplar: sub-skills become contract-bound verbs with zero flow knowledge, one flow-level Registry+Graph owns every edge, the coach renders commands from slots, and the companion stage folds into implement as a mode. Done means `check-flow-architecture.sh` exits 0 against the restructured skill and a fresh agent can assemble a **variant** flow from the pattern doc alone.

## Implementation

**Objective**: Ship the sub-skill pattern (doc + lints) and restructure the-flow to pass it — one phase, internally ordered A (pattern+lints, no self-edits) → B (Graph/Registry + fold + de-leak) → C (coach + views) → D (docs, verification, deploy last).
**Testing Strategy**: Lightweight (from spec) — the lint script is the verification layer; every removal task's Done-When is its grep; no mocks (resume checks use real state files + one synthetic fixture).
